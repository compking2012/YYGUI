Rectangle

  • Extends
  • YObject

new Rectangle()

Rectangle that represents by the coordinates of its 4 edges (left, top, right bottom). These fields can be accessed directly. Use width and height to retrieve the rectangle's width and height. Note that most methods do not check to see that the coordinates are sorted correctly (i.e. left <= right and top <= bottom).

Members

left :Number

the X coordinate of the left side of the rectangle.

top :Number

the Y coordinate of the top side of the rectangle.

right :Number

the X coordinate of the right side of the rectangle.

top :Number

the Y coordinate of the bottom side of the rectangle.

width :Number

the rectangle's width. This does not check for a valid rectangle (i.e. left <= right) so the result may be negative.

height :Number

the rectangle's height. This does not check for a valid rectangle (i.e. top <= bottom) so the result may be negative.

containsXY(x, y)

Indicates whether the specified point is inside this rectangle. The left and top are considered to be inside, while the right and bottom are not. This means that for a x,y to be contained: left <= x < right and top <= y < bottom. An empty rectangle never contains any point.

Parameters:

clone()

Creates and returns a copy of this Object. The default implementation returns a so-called "shallow" copy: It creates a new instance of the same class and then copies the field values (including object references) from this instance to the new instance. A "deep" copy, in contrast, would also recursively clone nested objects. A subclass that needs to implement this kind of cloning should call super's clone() to create the new instance and then create deep copies of the nested, mutable objects.
